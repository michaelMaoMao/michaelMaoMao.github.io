★ 里程碑版本详解

以下 15 个版本定义了 Claude Code 演进历程中的关键转折点 — 从平台化起点到多 Agent 协调，每个版本都是一个时代的缩影。

v2.1.x — 多 Agent 协作时代

2026.01 ~ 至今 | Agent View、/goal、Plugin 生态、企业级功能

v2.1.139
v2.1.139 2026-05-11
核心架构 重大功能

Agent View 研究预览版上线 — 运行 claude agents 统一查看所有会话（运行中/等待输入/已完成）。新增 /goal 命令设置完成条件，Claude 会持续工作直到条件满足。这是 Claude Code 从"单一对话"向"多 Agent 协调"演进的关键转折点。

  • Agent View：单屏管理所有 Claude Code 会话状态
  • /goal 命令：设定完成条件，支持交互模式、-p、Remote Control
  • /scroll-speed 命令调整鼠标滚轮速度
  • Hook args: string[] exec 模式，无需 shell 直接生成进程
  • Hook continueOnBlock 选项，允许 PostToolUse 拒绝后继续
  • MCP stdio 服务器接收 CLAUDE_PROJECT_DIR 环境变量
  • Compaction 提示现在要求保留敏感用户指令
v2.1.142 2026-05-14
重大功能 模型演进

Fast Mode 默认升级到 Opus 4.7（此前为 Opus 4.6）。Agent View 新增 8 个配置标志（--add-dir、--settings、--mcp-config 等），可为后台会话精确配置 MCP、权限和模型。

  • Fast Mode 默认使用 Opus 4.7（可通过 CLAUDE_CODE_OPUS_4_6_FAST_MODE_OVERRIDE=1 锁定 4.6）
  • Agent View 8 个新 CLI 标志精确控制后台会话
  • Plugin 根级 SKILL.md 自动识别为 Skill
  • /plugin 详情面板显示 LSP 服务器
  • 修复 MCP 工具超时上限仅 60 秒的问题
  • 修复 macOS 睡眠/唤醒导致后台会话消失
v2.1.143 2026-05-15
核心架构 插件系统

插件依赖强制执行：claude plugin disable 在有依赖时拒绝操作，enable 自动激活传递依赖。新增 worktree.bgIsolation: "none" 允许后台会话直接编辑工作目录。

  • 插件依赖链强制执行，阻止循环依赖
  • Marketplace 插件浏览面板显示预估 Token 消耗
  • worktree.bgIsolation: "none" 禁用 worktree 隔离
  • PowerShell 默认传递 -ExecutionPolicy Bypass
  • 后台会话跨唤醒保持模型和 effort 级别
  • Stop Hook 连续阻断 8 次后自动终止（防止死循环）
v2.1.147 2026-05-20
重大功能

/simplify 重命名为 /code-review，支持 effort 级别参数（如 /code-review high），报告正确性 Bug。--comment 参数可发布为 GitHub PR 行内评论。Pinned 后台会话空闲时保持存活。

  • /code-review 取代 /simplify，支持 effort 级别
  • --comment 参数发布 GitHub PR 行内评论
  • Pinned 后台会话空闲时保持存活，内存压力下优先释放非 Pinned
  • Diff 渲染性能优化（大型文件编辑）
  • 修复 PowerShell 权限绕过漏洞（cd..、cd\、cd~ 等内置 cd）
v2.1.149
/usage 分类明细、/diff 键盘滚动、GFM checkbox

/usage 新增分类使用明细（Skills、Subagents、Plugins、各 MCP 服务器成本），让你精确了解 Token 消耗去向。/diff 详情视图支持键盘滚动，GFM 任务列表渲染 checkbox。

  • /usage 按类别展示 Token 消耗明细
  • /diff 详情视图支持 j/k/箭头翻页
  • Markdown 渲染 GFM 任务列表 checkbox
  • Enterprise: allowAllClaudeAiMcps 管理设置
  • 修复多个权限绕过和安全加固（PowerShell、Git Worktree）
  • 修复 find 命令耗尽 macOS 文件描述符导致崩溃
v2.1.101 2026-04-10
重大功能 企业级 安全

新增 /team-onboarding 命令从本地使用生成团队上手指南。默认信任 OS CA 证书库（企业 TLS 代理开箱即用）。/ultraplan 自动配置云端环境。修复命令注入漏洞和长期会话内存泄漏。

  • /team-onboarding 生成团队上手指南
  • OS CA 证书库默认信任（企业 TLS 代理免配置）
  • 修复 LSP 二进制检测中的命令注入漏洞
  • 修复虚拟滚动器长期会话内存泄漏（数十份历史副本）
  • 修复 5 分钟硬编码请求超时限制
  • 修复 --resume 大型会话上下文丢失
v2.1.94 2026-04-07
平台支持 重大功能

Amazon Bedrock Mantle 支持上线。默认 effort 级别从 medium 提升至 high（API key、Bedrock/Vertex/Foundry、Team 和 Enterprise 用户）。CJK 流式输出修复。

  • Amazon Bedrock Mantle 支持（CLAUDE_CODE_USE_MANTLE=1）
  • 默认 effort 从 medium 提升至 high
  • Plugin skill 使用 frontmatter name 作为调用名
  • 修复 CJK/多字节文本流式输出 UTF-8 截断损坏
  • 修复 Console 登录 macOS Keychain 锁定导致静默失败
v2.1.72 2026-03-10
重大功能

新增 /loop 命令循环执行提示或斜杠命令（如 /loop 5m check the deploy）。Cron 调度工具支持会话内定时任务。Voice 按键自定义。Effort 级别简化为 low/medium/high。

  • /loop 定时循环执行提示
  • Cron 调度工具（CronCreate/CronDelete）
  • voice:pushToTalk 按键自定义（默认 Space，可设 meta+k）
  • Effort 简化为低/中/高，新符号
  • Native bash 命令解析模块（更快，无内存泄漏）
  • 修复长时间会话 stdin 冻结问题

Auto-compacting 变为即时操作。Agent 和 Bash 命令异步运行并支持唤醒消息。命名会话支持（/rename、/resume <name>）。.claude/rules/ 目录支持。这些特性将 Claude Code 从"一次性对话工具"转变为"持久化开发环境"。

  • Auto-compacting 即时完成

  • 图片尺寸元数据（大图精确坐标映射）
